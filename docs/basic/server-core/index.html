服务器核心入门

服务器服务器核心Bukkit

2026-02-05

选择正确的Minecraft服务器核心（服务端）是成功构建服务器的第一步。不同的服务端软件各有特色，适用于不同场景和需求。本指南将帮助你了解并选择最适合你的服务器核心，帮你踏出正确的第一步，拥有一个良好开端。

什么是服务器核心？

基础概念

服务器核心就是让Minecraft能够以多人游戏模式运行的程序文件。如果把Minecraft服务器比作一栋房子，那么服务器核心就是这栋房子的地基和框架。

简单来说就是：

  • 你平时玩的Minecraft客户端 = 单人游戏
  • 服务器核心 = 让多人可以同时进入同一个世界游戏的程序

核心的本质

服务器核心通常是一个 .jar 文件（Java程序），它包含了：

  • 游戏世界的运算逻辑
  • 玩家连接管理
  • 方块、生物、物品等游戏元素的处理
  • 指令系统
  • 插件/模组接口 部分核心

服务器核心的功能

主要作用

世界管理

生成地形、保存数据、处理区块加载

玩家管理

处理玩家登录、移动、互动

游戏逻辑

计算伤害、生物AI、红石电路等

网络通信

接收客户端数据、发送世界更新

扩展支持

提供插件或模组的运行环境 部分核心

为什么我需要选择核心？

如果你没接触过架设MC服务器，你可能会问，为什么我需要在服务器核心这一方面做选择？难道官方的服务器核心不好吗？

Mojang官方确实提供了服务器核心（Vanilla），但它的功能有限，因此Minecraft社区开发了各种各样的“增强版”核心：

  • 更好的性能优化
  • 插件支持(添加各种功能)
  • 模组支持(改变游戏玩法)
  • 反作弊功能
  • 更多管理工具

难不成你还想用这核心？

主流服务器核心类型

1. Vanilla（官方原版服务端）

简介

这是一个由Mojang官方发布的核心，提供100%的原版游戏体验，但是不支持插件和模组，而且性能也一般

适用场景

  • 纯原版生存服务器
  • 小型私人服务器 朋友联机
  • 不需要任何扩展功能的服务器

优点

最稳定、最纯粹

缺点

功能少、性能差、无法防作弊

2. Bukkit系核心（插件服）

插件服就是能装插件的服务器，插件介绍可以参考这一节：插件/模组入门

而Bukkit系核心是当今最流行的插件服核心体系，经过多年发展已经非常成熟。

2.1 CraftBukkit 淘汰

简介

这是最早的插件服核心，它支持Bukkit插件，但现在基本被淘汰你猜猜怎么淘汰的

现状：已经很少使用，被后继者（如Spigot）取代

2.2 Spigot

简介

这是CraftBukkit核心的优化版分支，支持Bukkit和Spigot插件，且有一定的性能优化，相关社区也很活跃，版本更新也比较及时

适用场景

  • 中小型插件服务器
  • 需要基础插件功能
  • 配置要求不高的服务器

优点

  • 插件生态成熟
  • 稳定性好
  • 文档丰富

缺点

  • 性能优化不如Paper
  • 大型服务器可能卡顿

官网：https://www.spigotmc.org/

2.3 Paper 推荐

简介

这是Spigot核心的高性能分支核心，是目前最流行的插件服核心，包含大量性能优化和bug修复（给原版特新也修复了），且完全兼容Spigot/Bukkit插件，不仅如此，它还提供额外的API

适用场景

  • 中大型插件服务器
  • 需要高性能的服务器
  • 对稳定性有要求的服务器

优点

  • 性能优秀
  • 开发团队专业
  • 更新快速
  • 修复了很多原版bug

缺点

  • 部分插件可能不兼容 极少数
  • 某些机制与原版略有不同（因为它修复了原版的特性）

官网：https://papermc.io/

2.4 Purpur

简介

这是Paper的分支核心，它提供更多游戏性配置选项，性能优化也更加激进，且可自定义的功能更多

适用场景

  • 需要高度自定义的服务器
  • 追求极致性能的服务器

优点

  • 配置选项极其丰富
  • 性能优于Paper

缺点

  • 某些优化可能影响原版特性
  • 相对小众

官网：https://purpurmc.org/

3. 模组服核心

模组服核心大家都应该比较熟悉，它就是客户端里面的那些模组加载器，模组服核心允许加载模组（Mod），改变游戏玩法。

3.1 Forge

简介

大家再也熟悉不过的经典的模组加载器，它支持绝大多数Mod，生态系统也是最庞大的

适用场景

  • 需要运行大型整合包的服务器
  • 科技、魔法、冒险类模组服

优点

  • 模组数量最多
  • 文档和教程丰富

缺点

  • 性能较差
  • 启动速度慢

官网：https://files.minecraftforge.net/

3.2 Fabric

简介

Fabric是新一代轻量级的模组加载器，它启动快、性能好，而且模组生态也在快速发展

适用场景

  • 小型模组服
  • 需要快速更新到新版本的模组服
  • 注重性能的模组服

优点

  • 性能优秀
  • 更新速度快
  • 代码现代化

缺点

  • 模组数量不如Forge
  • 部分老模组不支持

官网：https://fabricmc.net/

3.3 NeoForge

简介

这是Forge的现代化分支，它改进了Forge的很多问题，且逐渐获得模组开发者支持，但是它的生态目前依然比不过Forge这样的老牌模组加载器

适用场景

  • 新版本模组服
  • 追求更好性能的Forge用户

现状：正在快速发展中

官网：https://neoforged.net/

4. 混合核心（插件+模组）

混合核心，顾名思义，这类核心同时支持插件和模组，但兼容性可能存在问题。

4.1 Mohist 不推荐

简介

这是Forge + Paper的混合核心，它同时支持Mod和插件，但是稳定性非常差……

适用场景

  • 既需要模组又需要插件功能

优点

  • 功能全面

缺点

  • 兼容性问题多
  • 可能不稳定
  • Bug较多

警告

不推荐新手使用,问题排查困难

官网：https://mohistmc.cn/resources/mohist

4.2 CatServer 停更

简介

一个由国人开发的混合端核心，结合了Forge + Spigot，只可惜这个项目已经停更

现状：已停止维护

官网：https://catmc.org/

4.3 Arclight

简介

这是一个Forge/Fabric + Bukkit的混合核心，它支持多个版本

适用场景

  • 模组+插件混合服

注意

兼容性问题需要仔细测试

官网：https://arclight.izzel.io/

5. 基岩版核心 拓展

用于Minecraft基岩版（手机版、Win10版）的服务器。本来这一篇是主要讲解Java版核心的，不过为了拓宽你们的视野，这里特意额外提一嘴基岩版的核心。

5.1 BDS 官方

简介

这是官方基岩版服务端，它并不支持插件

5.2 Nukkit系列

简介

这是基岩版的插件服核心，使用 Java 编写，它支持基岩版插件，但它现已停止维护，现在更新的基本是Nukkit的分支核心，你可以自行搜索相关服务端软件

如何选择合适的核心

决策流程图

想开什么类型的服务器?
│
├─ 纯原版生存 → Vanilla 或 Paper(优化更好)
│
├─ 插件服(小游戏、RPG、生存辅助等)
│   │
│   ├─ 小型服务器(10人以下) → Spigot
│   │
│   └─ 中大型服务器 → Paper(推荐) 或 Purpur
│
├─ 模组服(科技、魔法、冒险等)
│   │
│   ├─ 大型整合包 → Forge
│   │
│   └─ 轻量模组/新版本 → Fabric
│
└─ 插件+模组混合 → Mohist/Arclight(谨慎使用)

注

如果你比比较熟悉这些核心，那么你完全可以自行决定，不必死板地按照上面的决策树来选择你的服务器核心

选择建议

新手服主优先选择顺序：

  1. Paper - 适合90%的插件服需求
  2. Fabric - 轻量模组服首选
  3. Forge - 大型整合包必选
  4. Purpur - 高手服主，需要更多自定义
  5. Spigot - Paper的备选方案

不要选择以下核心！

  • 混合核心（除非你很清楚自己在做什么）
  • 已停更的核心
  • 来源不明的核心 不推荐

核心的下载与安装

准备工作

  1. 确认游戏版本

    首先确定你要开的服务器版本（如1.20.1），核心版本必须与客户端版本一致。

  2. 安装Java

    不同Minecraft版本需要不同的Java版本：

    Minecraft版本所需Java版本
    1.12-1.16.5Java 8
    1.17-1.17.1Java 16
    1.18-1.20.4Java 17
    1.20.5-1.21.11Java 21
    26.1+Java 25

    Java下载：

    • 你需要下载对应版本的JDK（Java Development Kit），这里给出两个选项：
    • Adoptium（OpenJDK）：https://adoptium.net/
    • Oracle（甲骨文）：https://www.oracle.com/cn/java/technologies/downloads/

  3. 使用检查Java版本的命令行检测版本：

    java -version

下载核心

  1. Paper下载

    1. 访问官网：https://papermc.io/downloads
    2. 选择游戏版本(如1.20.1)
    3. 点击下载最新构建版本
    4. 得到文件如：paper-1.20.1-196.jar

  2. Fabric下载

    1. 访问官网：https://fabricmc.net/use/server/
    2. 选择游戏版本
    3. 下载Fabric服务端安装器
    4. 运行安装器,选择安装路径
    5. 生成服务端文件

  3. Forge下载

    1. 访问：https://files.minecraftforge.net/
    2. 选择Minecraft版本
    3. 下载Installer(安装器)
    4. 运行安装器:
      java -jar forge-installer.jar --installServer
    5. 等待安装完成

  4. 更多核心下载链接可以自行搜索……

安装步骤

对于Paper/Spigot等一类的服务器核心，你可以直接用启动脚本运行：

详情见：启动脚本入门

  1. 创建服务器文件夹：

    新建文件夹，如：MyServer

  2. 放入核心文件：

    将下载的 .jar 文件放入文件夹
重命名为 server.jar (可选，便于管理)

  3. 首次启动：

    Windows系统创建 start.bat：

    start.bat
    @echo off
java -Xms2G -Xmx4G -jar server.jar nogui
pause

    Linux系统创建 start.sh：

    start.sh
    #!/bin/bash
java -Xms2G -Xmx4G -jar server.jar nogui

  4. 同意EULA：

    首次运行会生成 eula.txt，打开文件：

    eula.txt
    eula=false（将false改为true）

  5. 再次启动：

    运行启动脚本，服务器开始生成世界

参数说明

java -Xms2G -Xmx4G -jar server.jar nogui
  • java：调用Java运行环境
  • -Xms2G：初始内存分配2GB
  • -Xmx4G：最大内存分配4GB
  • -jar server.jar：运行的核心文件
  • nogui：不启动图形界面 推荐

内存分配建议：

服务器规模最小内存推荐内存
1-5人1G2G
5-10人2G4G
10-20人4G6G
20-50人6G8G+
50人以上8G16G+

核心的配置与优化

原版配置

server.properties

这是服务器的主配置文件，以下是部分重要参数：

server.properties
# 服务器端口(默认25565)
server-port=25565

# 最大玩家数
max-players=20

# 游戏模式(survival/creative/adventure)
gamemode=survival

# 难度(peaceful/easy/normal/hard)
difficulty=normal

# 是否开启正版验证
online-mode=true

# 服务器名称
motd=我的Minecraft服务器

# 视距(区块加载距离)
view-distance=10

# 世界种子
level-seed=

# 是否允许飞行
allow-flight=false

# 是否生成生物
spawn-monsters=true
spawn-animals=true

# 白名单
white-list=false

重要配置说明：重要

  1. online-mode:

    • true: 正版验证，只有正版玩家能进入
    • false: 离线模式，离线玩家也能进

  2. view-distance:

    • 越大越消耗性能
    • 建议: 6-10（普通服务器），4-6（性能较差）

  3. max-players:

    • 根据服务器配置设置
    • 不要设置过高导致卡顿

Paper优化配置

警告

以下配置项虽然说可以在一定程度上优化性能，但是可能破坏原版的特性，导致一些红石机械无法使用，请谨慎使用以下优化配置！

# 实体优化

  activation-range:
    animals: 32
    monsters: 32
    raiders: 48
    misc: 16
    
  # 行为优化

    # 僵尸破门
    door-breaking-difficulty: hard

paper-world.yml

paper-world.yml
# 生物生成优化

  # 每tick生成尝试次数
  monsters: 1

  water-animals: 1

  # 禁用移动事件(提升性能)

性能优化

1. JVM参数优化

详情见：启动脚本入门

你可以使用Aikar's Flags启动参数作为优化项：

java -Xms4G -Xmx4G -XX:+UseG1GC -XX:+ParallelRefProcEnabled \
-XX:MaxGCPauseMillis=200 -XX:+UnlockExperimentalVMOptions \
-XX:+DisableExplicitGC -XX:+AlwaysPreTouch \
-XX:G1NewSizePercent=30 -XX:G1MaxNewSizePercent=40 \
-XX:G1HeapRegionSize=8M -XX:G1ReservePercent=20 \
-XX:G1HeapWastePercent=5 -XX:G1MixedGCCountTarget=4 \
-XX:InitiatingHeapOccupancyPercent=15 \
-XX:G1MixedGCLiveThresholdPercent=90 \
-XX:G1RSetUpdatingPauseTimePercent=5 \
-XX:SurvivorRatio=32 -XX:+PerfDisableSharedMem \
-XX:MaxTenuringThreshold=1 -Dusing.aikars.flags=https://mcflags.emc.gs \
-Daikars.new.flags=true -jar server.jar nogui

注

Xms 和 Xmx 需要设置为相同值

2. 插件优化

相比之下，插件优化就要简单许多，这应该是许多服主都会的操作：

  1. 定期清理一次插件，删除不用的插件
  2. 不要使用优化跟一坨💩一样的插件！否则你的服务器就……
  3. 可以试着使用Spark（火花）插件/模组分析性能
  4. 功能重复的插件只装一个，不要重复安装

提示

这里推荐大家使用Spark作为分析性能的插件/模组，这款插件真的非常好用！而且它还被Paper服务端捆绑安装在服务器上

3. 世界优化

  • 预生成地图：使用Chunky插件预生成世界

    /chunky radius 5000
/chunky start

  • 定期清理实体：

    • 掉落物清理插件
    • 定期清理无用实体

  • 限制红石：

    • 使用反卡服插件限制红石机器
    • 检测并移除卡服装置

4. 数据库优化 大型服务器

  • 使用MySQL而非SQLite
  • 定期清理过期数据
  • 优化查询语句

备份策略

自动备份

你可以使用备份插件或脚本定期备份：

Linux备份脚本示例：

#!/bin/bash
DATE=$(date +%Y-%m-%d_%H-%M-%S)
BACKUP_DIR="/backups"
SERVER_DIR="/home/minecraft"

# 创建备份
tar -czf $BACKUP_DIR/backup_$DATE.tar.gz $SERVER_DIR/world

# 删除7天前的备份
find $BACKUP_DIR -name "backup_*.tar.gz" -mtime +7 -delete

echo "备份完成: backup_$DATE.tar.gz"

备份频率建议：

  • 世界数据: 每6-12小时
  • 插件配置: 每天
  • 完整备份: 每周

常见问题解答（FAQ）

1. 服务器启动失败

问题描述：双击启动文件闪退

解决方案：

  • 检查Java版本是否匹配
  • 查看logs文件夹中的日志
  • 确认内存分配是否合理
  • 检查eula.txt是否已同意

2. 无法连接服务器

问题描述：玩家无法进入服务器

解决方案：

  • 检查防火墙是否开放25565端口
  • 确认服务器IP地址是否正确
  • 查看server.properties中的端口设置
  • 检查路由器端口转发(外网访问)
  • 确认游戏版本是否一致

3. 服务器卡顿

详情见：卡顿入门

问题描述：TPS（每秒刻数）低于20

解决方案：

  • 使用Spark分析性能瓶颈
  • 减少视距（view-distance）
  • 清理过多实体
  • 优化红石机器
  • 预生成世界
  • 检查插件/模组是否有问题
  • 升级服务器配置

4. 插件冲突

问题描述：安装插件后出现报错

解决方案：

  • 查看日志中的错误信息
  • 确认插件版本与核心版本匹配
  • 检查是否有功能冲突的插件
  • 逐个停用插件排查问题
  • 更新插件到最新版本

5. 内存溢出

问题描述：服务器（终端）提示“Out of Memory”

解决方案：

  • 增加-Xmx参数的内存分配
  • 减少加载的区块数量
  • 清理过期的世界备份
  • 优化插件配置
  • 考虑升级服务器硬件

6. 版本更新

问题描述：如何升级服务器版本

步骤：

  1. 完整备份当前服务器
  2. 下载新版本核心
  3. 替换旧的核心文件
  4. 检查插件兼容性，更新插件
  5. 启动服务器测试
  6. 如有问题，恢复备份

注意：

  • 不要跨越太多版本更新
  • 某些版本更新会改变世界格式
  • 务必备份后再更新

7. 模组服特有问题

问题描述: 模组无法加载

解决方案:

  • 确认模组版本与Forge/Fabric版本匹配
  • 检查模组依赖是否安装
  • 查看crash日志
  • 确认模组之间无冲突
  • 使用整合包管理工具（如CurseForge）
